# AgentInfo

<APILink fn="mlflow.types.agent_info.AgentInfo">`AgentInfo`</APILink> is a Pydantic `BaseModel` that describes your agent's capabilities and interface. It is passed to the <APILink fn="mlflow.genai.agent_server.AgentServer">`AgentServer`</APILink> constructor and served at the `GET /agent/info` endpoint, enabling runtime discovery by clients, monitoring tools, and orchestration platforms.

## Why Use AgentInfo?

When agents are deployed behind a uniform `/invocations` endpoint, external systems need a way to discover _what_ each agent does without calling it. `AgentInfo` provides that discovery layer:

- **Service catalogs** can index agents by name, description, and tags
- **Orchestrators** can route requests based on `use_case` or `agent_api`
- **Monitoring dashboards** can display agent version and MLflow version
- **Clients** can inspect `custom_inputs_schema` / `custom_outputs_schema` to build dynamic UIs

## Quick Start

```python
from mlflow.genai.agent_server import AgentServer, AgentInfo

agent_info = AgentInfo(
    name="math-tutor",
    description="Helps students solve math problems with step-by-step explanations",
    version="1.2.0",
    tags={"team": "education", "env": "production"},
)

agent_server = AgentServer("ResponsesAgent", agent_info=agent_info)
```

```bash
curl http://localhost:8000/agent/info
```

```json
{
  "name": "math-tutor",
  "use_case": "agent",
  "mlflow_version": "3.6.0",
  "agent_api": "responses",
  "description": "Helps students solve math problems with step-by-step explanations",
  "version": "1.2.0",
  "tags": {"team": "education", "env": "production"}
}
```

## Fields Reference

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | `"mlflow_agent_server"` | Agent name. When no `AgentInfo` is provided to `AgentServer`, defaults to the `DATABRICKS_APP_NAME` environment variable. |
| `use_case` | `str` | `"agent"` | Use case category. |
| `mlflow_version` | `str` | `mlflow.__version__` | MLflow version running the server. Automatically populated. |
| `agent_api` | `str \| None` | `None` | Agent API protocol (e.g. `"responses"`). Auto-set to `"responses"` when `agent_type="ResponsesAgent"`. |
| `description` | `str \| None` | `None` | Human-readable description of what the agent does. |
| `version` | `str \| None` | `None` | Version string for the agent (e.g. `"1.0.0"`). |
| `metadata` | `dict[str, Any] \| None` | `None` | Extensible dict for additional information. Reserved keys: `custom_inputs_schema`, `custom_outputs_schema`. |
| `tags` | `dict[str, str] \| None` | `None` | Arbitrary key-value pairs for categorization and filtering. |

Fields set to `None` are omitted from the `/agent/info` response via `model_dump(exclude_none=True)`.

## Custom Input/Output Schemas

If your agent accepts or returns fields beyond the standard interface, declare their JSON schemas in the `metadata` dict using the reserved keys `custom_inputs_schema` and `custom_outputs_schema`. These schemas let clients understand the agent's extended interface without trial and error.

```python
agent_info = AgentInfo(
    name="research-assistant",
    description="Searches internal knowledge base and summarizes findings",
    version="2.0.0",
    metadata={
        "custom_inputs_schema": {
            "type": "object",
            "properties": {
                "search_depth": {
                    "type": "integer",
                    "description": "Maximum number of documents to search",
                },
                "language": {
                    "type": "string",
                    "enum": ["en", "es", "fr"],
                    "description": "Response language",
                },
            },
        },
        "custom_outputs_schema": {
            "type": "object",
            "properties": {
                "sources": {
                    "type": "array",
                    "items": {"type": "string"},
                    "description": "URLs of referenced documents",
                },
                "confidence": {
                    "type": "number",
                    "description": "Confidence score between 0 and 1",
                },
            },
        },
    },
)
```

## Subclassing

`AgentInfo` is a standard Pydantic `BaseModel`, so you can subclass it to add domain-specific fields. Subclass fields are included in the `/agent/info` response alongside the standard fields.

```python
from mlflow.types.agent_info import AgentInfo


class MyAgentInfo(AgentInfo):
    capabilities: list[str] = []
    max_context_tokens: int = 128000


agent_info = MyAgentInfo(
    name="code-reviewer",
    description="Reviews pull requests and suggests improvements",
    capabilities=["code-review", "refactoring", "security-audit"],
    max_context_tokens=200000,
)
```

This returns:

```json
{
  "name": "code-reviewer",
  "use_case": "agent",
  "mlflow_version": "3.6.0",
  "description": "Reviews pull requests and suggests improvements",
  "capabilities": ["code-review", "refactoring", "security-audit"],
  "max_context_tokens": 200000
}
```

## Default Behavior

If you don't pass an `AgentInfo` to `AgentServer`, one is created automatically:

```python
# These two are equivalent:
AgentServer("ResponsesAgent")
AgentServer("ResponsesAgent", agent_info=AgentInfo())
```

The default `/agent/info` response:

```json
{
  "name": "mlflow_agent_server",
  "use_case": "agent",
  "mlflow_version": "3.6.0",
  "agent_api": "responses"
}
```

**Environment variable**: When `DATABRICKS_APP_NAME` is set and no `AgentInfo` is provided, its value is used as the default `name`.

**Automatic `agent_api`**: When `agent_type="ResponsesAgent"`, the `agent_api` field is automatically set to `"responses"` unless you explicitly provide a different value.

## Full Example

Here is a complete example showing `AgentInfo` used with an `AgentServer` that hosts a Responses API agent:

**`agent.py`** — Define the agent logic:

```python
from agents import Agent, Runner
from mlflow.genai.agent_server import invoke, stream
from mlflow.types.responses import ResponsesAgentRequest, ResponsesAgentResponse

agent = Agent(
    name="Math Tutor",
    instructions="You provide help with math problems. Explain your reasoning and include examples",
)


@invoke()
async def non_streaming(request: ResponsesAgentRequest) -> ResponsesAgentResponse:
    msgs = [i.model_dump() for i in request.input]
    result = await Runner.run(agent, msgs)
    return ResponsesAgentResponse(output=[item.to_input_item() for item in result.new_items])
```

**`start_server.py`** — Configure and start the server with `AgentInfo`:

```python
import agent  # noqa: F401
from mlflow.genai.agent_server import AgentInfo, AgentServer

agent_info = AgentInfo(
    name="math-tutor",
    description="Helps students solve math problems with step-by-step explanations",
    version="1.0.0",
    tags={"team": "education", "subject": "math"},
    metadata={
        "custom_inputs_schema": {
            "type": "object",
            "properties": {
                "difficulty": {
                    "type": "string",
                    "enum": ["easy", "medium", "hard"],
                    "description": "Problem difficulty level",
                },
            },
        },
    },
)

agent_server = AgentServer("ResponsesAgent", agent_info=agent_info)
app = agent_server.app


def main():
    agent_server.run(app_import_string="start_server:app")


if __name__ == "__main__":
    main()
```

Start the server and query the info endpoint:

```bash
python3 start_server.py --reload

# In another terminal:
curl http://localhost:8000/agent/info | python3 -m json.tool
```
