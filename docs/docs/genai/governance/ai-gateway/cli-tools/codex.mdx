# ⚡ OpenAI Codex + MLflow AI Gateway

Route [OpenAI Codex Desktop](https://github.com/openai/codex) through the MLflow AI Gateway to get centralized tracing and observability — while each developer authenticates with their own OpenAI subscription.

## Prerequisites

- MLflow server running with a SQL backend (`mlflow server --port 5000`)
- An OpenAI endpoint configured in the gateway (see [Create an Endpoint](/genai/governance/ai-gateway/endpoints/create-and-manage))
- Codex Desktop installed

## Step 1: Create an OpenAI Endpoint

In the MLflow UI at `http://localhost:5000/#/settings`, go to **Endpoints** and create an endpoint:

- **Provider:** OpenAI
- **Model:** `gpt-4o` (or your preferred model)
- **LLM Connection:** Select an existing connection, or leave blank to rely on client credentials

:::tip
You can leave the server-side API key empty if you want all users to authenticate with their own OpenAI subscriptions. The gateway will use the credential supplied in each request.
:::

## Step 2: Point Codex at the Gateway

Set the `OPENAI_BASE_URL` environment variable to the gateway's OpenAI passthrough endpoint:

```bash
export OPENAI_BASE_URL="http://localhost:5000/gateway/openai/v1"
```

Codex Desktop reads this variable automatically and routes all API calls through the gateway.

## Step 3: Run Codex

```bash
codex
```

Codex will authenticate using your existing OpenAI credentials and all requests will be proxied through the gateway.

## What You Get

Every session is captured as an MLflow trace. Open the **Tracing** tab in the MLflow UI to inspect inputs, outputs, token usage, and latency for every request.
