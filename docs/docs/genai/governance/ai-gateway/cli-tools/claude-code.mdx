# 🤖 Claude Code + MLflow AI Gateway

Route [Claude Code](https://docs.anthropic.com/en/docs/claude-code) through the MLflow AI Gateway to get centralized tracing and observability — while each developer authenticates with their own Anthropic subscription.

## Prerequisites

- MLflow server running with a SQL backend (`mlflow server --port 5000`)
- An Anthropic endpoint configured in the gateway (see [Create an Endpoint](/genai/governance/ai-gateway/endpoints/create-and-manage))
- Claude Code installed (`npm install -g @anthropic-ai/claude-code`)

## Step 1: Create an Anthropic Endpoint

In the MLflow UI at `http://localhost:5000/#/settings`, go to **Endpoints** and create an endpoint:

- **Provider:** Anthropic
- **Model:** `claude-opus-4-5` (or your preferred model)
- **LLM Connection:** Select an existing connection, or leave blank to rely on client credentials

:::tip
You can leave the server-side API key empty if you want all users to authenticate with their own Anthropic subscriptions. The gateway will use the credential supplied in each request.
:::

## Step 2: Point Claude Code at the Gateway

Set the `ANTHROPIC_BASE_URL` environment variable to the gateway's Anthropic passthrough endpoint:

```bash
export ANTHROPIC_BASE_URL="http://localhost:5000/gateway/anthropic/v1"
```

Claude Code reads this variable automatically and routes all API calls through the gateway.

## Step 3: Run Claude Code

```bash
claude
```

Claude Code will authenticate using your existing Anthropic credentials (stored in `~/.claude`) and all requests will be proxied through the gateway.

## What You Get

Every conversation is captured as an MLflow trace. Open the **Tracing** tab in the MLflow UI to inspect inputs, outputs, token usage, and latency for every request.
