# ![Gemini](/images/logos/google-gemini-logo.svg) Gemini CLI + MLflow AI Gateway

Route [Gemini CLI](https://github.com/google-gemini/gemini-cli) through the MLflow AI Gateway to get centralized tracing and observability — while each developer authenticates with their own Google subscription.

## Prerequisites

- MLflow server running with a SQL backend (`mlflow server --port 5000`)
- Gemini CLI installed (`npm install -g @google/gemini-cli`)

## Step 1: Create a Gemini Endpoint

Navigate to the **AI Gateway** tab at `http://localhost:5000/#/gateway` and click **Create Endpoint**.

- **Provider:** Gemini
- **Model:** `gemini-2.0-flash` (or your preferred model)
- **Endpoint name:** choose a name, e.g. `my-gemini-endpoint`
- **Authentication:** select **API Key** based authentication
- **LLM Connection:** select an existing connection (see [Create an LLM Connection](/genai/governance/ai-gateway/api-keys/create-and-manage))

:::tip
**Subscription passthrough:** If your users run Gemini CLI with their own Google subscription, you can create the LLM Connection with a dummy API key (e.g. `dummy`). The gateway detects Gemini CLI's `User-Agent` and forwards the client's own credentials instead of the server-side key.

**Server-side API key:** You can also create the LLM Connection with a real Gemini API key. In this case all requests are billed to that key regardless of who is running Gemini CLI.

Both modes are supported — choose based on your team's billing and access control preferences.
:::

## Step 2: Configure Environment Variables

Set the following environment variables so Gemini CLI routes through the gateway and uses your endpoint:

```bash
export GOOGLE_GEMINI_BASE_URL="http://localhost:5000/gateway/gemini"
export GEMINI_MODEL="my-gemini-endpoint"   # your endpoint name from Step 1
export GEMINI_API_KEY="your-google-api-key"
```

## Step 3: Run Gemini CLI

```bash
gemini
```

Gemini CLI authenticates using your existing Google credentials and all requests are proxied through the gateway.

## What You Get

Every conversation is captured as an MLflow trace. Open the **Tracing** tab in the MLflow UI to inspect inputs, outputs, token usage, and latency for every request.
