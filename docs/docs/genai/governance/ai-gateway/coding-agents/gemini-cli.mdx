# ✨ Gemini CLI + MLflow AI Gateway

Route [Gemini CLI](https://github.com/google-gemini/gemini-cli) through the MLflow AI Gateway to get centralized tracing and observability — while each developer authenticates with their own Google subscription.

## Prerequisites

- MLflow server running with a SQL backend (`mlflow server --port 5000`)
- A Gemini endpoint configured in the gateway (see [Create an Endpoint](/genai/governance/ai-gateway/endpoints/create-and-manage))
- Gemini CLI installed (`npm install -g @google/gemini-cli`)

## Step 1: Create a Gemini Endpoint

In the MLflow UI at `http://localhost:5000/#/settings`, go to **Endpoints** and create an endpoint:

- **Provider:** Gemini
- **Model:** `gemini-2.0-flash` (or your preferred model)
- **LLM Connection:** Select an existing connection, or leave blank to rely on client credentials

:::tip
You can leave the server-side API key empty if you want all users to authenticate with their own Google subscriptions. The gateway will use the credential supplied in each request.
:::

## Step 2: Point Gemini CLI at the Gateway

Set the `GEMINI_API_BASE_URL` environment variable to the gateway's Gemini passthrough endpoint:

```bash
export GEMINI_API_BASE_URL="http://localhost:5000/gateway/gemini/v1beta"
```

Gemini CLI reads this variable automatically and routes all API calls through the gateway.

## Step 3: Run Gemini CLI

```bash
gemini
```

Gemini CLI will authenticate using your existing Google credentials and all requests will be proxied through the gateway.

## What You Get

Every conversation is captured as an MLflow trace. Open the **Tracing** tab in the MLflow UI to inspect inputs, outputs, token usage, and latency for every request.
