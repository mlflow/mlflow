# ![Claude Code](/images/logos/claude-code-logo.png) Claude Code + MLflow AI Gateway

Route [Claude Code](https://docs.anthropic.com/en/docs/claude-code) through the MLflow AI Gateway to get centralized tracing and observability — while each developer authenticates with their own Anthropic subscription.

## Prerequisites

- MLflow server running with a SQL backend (`mlflow server --port 5000`)
- Claude Code installed (`npm install -g @anthropic-ai/claude-code`)

## Step 1: Create an Anthropic Endpoint

Navigate to the **AI Gateway** tab at `http://localhost:5000/#/gateway` and click **Create Endpoint**.

- **Provider:** Anthropic
- **Model:** `claude-opus-4-5` (or your preferred model)
- **Endpoint name:** choose a name, e.g. `my-claude-endpoint`
- **LLM Connection:** select an existing connection (see [Create an LLM Connection](/genai/governance/ai-gateway/api-keys/create-and-manage))

:::tip
**Subscription passthrough:** If your users run Claude Code with their own Anthropic subscription, you can create the LLM Connection with a dummy API key (e.g. `dummy`). The gateway detects Claude Code's `User-Agent` and forwards the client's own credentials instead of the server-side key.

**Server-side API key:** You can also create the LLM Connection with a real Anthropic API key. In this case all requests are billed to that key regardless of who is running Claude Code.

Both modes are supported — choose based on your team's billing and access control preferences.
:::

## Step 2: Configure Environment Variables

Set the following environment variables so Claude Code routes through the gateway and uses your endpoint:

```bash
export ANTHROPIC_BASE_URL="http://localhost:5000/gateway/anthropic/v1"
export ANTHROPIC_MODEL="my-claude-endpoint"   # your endpoint name from Step 1
```

Alternatively, you can pass the model on the command line instead of setting `ANTHROPIC_MODEL`:

```bash
claude --model my-claude-endpoint
```

## Step 3: Run Claude Code

```bash
claude
```

Claude Code authenticates using your existing Anthropic credentials (stored in `~/.claude`) and all requests are proxied through the gateway.

## What You Get

Every conversation is captured as an MLflow trace. Open the **Tracing** tab in the MLflow UI to inspect inputs, outputs, token usage, and latency for every request.
