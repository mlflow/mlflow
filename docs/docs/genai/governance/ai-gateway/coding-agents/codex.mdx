# ![OpenAI](/images/logos/openai-logo-only.png) OpenAI Codex + MLflow AI Gateway

Route [OpenAI Codex](https://github.com/openai/codex) through the MLflow AI Gateway to get centralized tracing and observability — while each developer authenticates with their own OpenAI subscription.

## Prerequisites

- MLflow server running with a SQL backend (`mlflow server --port 5000`)
- Codex installed (`npm install -g @openai/codex`)

## Step 1: Create an OpenAI Endpoint

Navigate to the **AI Gateway** tab at `http://localhost:5000/#/gateway` and click **Create Endpoint**.

- **Provider:** OpenAI
- **Model:** `gpt-4o` (or your preferred model)
- **Endpoint name:** choose a name, e.g. `my-codex-endpoint`
- **LLM Connection:** select an existing connection (see [Create an LLM Connection](/genai/governance/ai-gateway/api-keys/create-and-manage))

:::tip
**Subscription passthrough:** If your users run Codex with their own OpenAI subscription, you can create the LLM Connection with a dummy API key (e.g. `dummy`). The gateway detects Codex's `User-Agent` and forwards the client's own credentials instead of the server-side key.

**Server-side API key:** You can also create the LLM Connection with a real OpenAI API key. In this case all requests are billed to that key regardless of who is running Codex.

Both modes are supported — choose based on your team's billing and access control preferences.
:::

## Step 2: Configure Environment Variables

Set the following environment variable so Codex routes through the gateway:

```bash
export OPENAI_BASE_URL="http://localhost:5000/gateway/openai/v1"
```

## Step 3: Run Codex

Pass your endpoint name as the model:

```bash
codex --model my-codex-endpoint
```

Codex authenticates using your existing OpenAI credentials and all requests are proxied through the gateway.

## What You Get

Every session is captured as an MLflow trace. Open the **Tracing** tab in the MLflow UI to inspect inputs, outputs, token usage, and latency for every request.
