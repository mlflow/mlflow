# Tracing Hermes Agent

[MLflow Tracing](/genai/tracing) can capture end-to-end [Hermes Agent](https://github.com/NousResearch/hermes-agent) execution traces, including LLM calls, tool invocations, and long-running agent sessions, by ingesting OpenTelemetry spans emitted by the [`hermes-otel`](https://github.com/briancaffey/hermes-otel) plugin.

<ImageBox
  src="/images/llms/hermes/hermes-agent-trace-detail.png"
  alt="Hermes Agent tool-calling trace in the MLflow UI"
/>

Once configured, Hermes traces appear in MLflow with:

- An `AGENT` root span for each Hermes turn
- `LLM` spans for model calls
- `TOOL` spans for terminal and tool execution
- Token usage, latency, and session metadata

## Setup

### Requirements

- MLflow tracking server running
- Hermes Agent installed
- The [`hermes-otel`](https://github.com/briancaffey/hermes-otel) plugin installed into the Hermes environment

### Step 1: Create an MLflow Experiment

Create or choose an MLflow experiment for Hermes traces:

```python
import mlflow

mlflow.set_tracking_uri("http://localhost:5000")
experiment = mlflow.get_experiment_by_name("hermes-agent")
if experiment is None:
    experiment_id = mlflow.create_experiment("hermes-agent")
else:
    experiment_id = experiment.experiment_id

print(experiment_id)
```

### Step 2: Install Hermes OTLP Tracing

Install the Hermes OpenTelemetry plugin:

```bash
hermes plugins install briancaffey/hermes-otel
```

Then install the plugin package into the same Python environment Hermes itself uses:

```bash
~/git/hermes-agent/venv/bin/pip install -e ~/.hermes/plugins/hermes_otel
```

### Step 3: Point the Plugin at MLflow

Create `~/.hermes/plugins/hermes_otel/config.yaml` and point it at the MLflow OTLP endpoint:

```yaml
enabled: true
force_flush_on_session_end: true
capture_sender_id: true
capture_conversation_history: true

backends:
  - type: otlp
    name: mlflow
    endpoint: http://localhost:5000/v1/traces
    metrics: false
    logs: false
    headers:
      x-mlflow-experiment-id: "<your-experiment-id>"
```

:::tip
`x-mlflow-experiment-id` is required for OTLP ingestion. MLflow uses it to route incoming Hermes spans to the right experiment.
:::

### Step 4: Enable the Plugin in Hermes

Update `~/.hermes/config.yaml` so Hermes loads the plugin and uses the model/provider you want:

```yaml
model:
  default: "gpt-4o-mini"
  provider: "custom"
  base_url: "http://localhost:5000/gateway/mlflow/v1"

plugins:
  enabled:
    - hermes_otel
```

The `base_url` above is optional. If you want centralized routing, guardrails, and cost tracking, point Hermes at the [MLflow AI Gateway](/genai/governance/ai-gateway/). If you only want tracing, you can point Hermes at any supported provider and keep `hermes-otel` exporting spans directly to MLflow.

If you use the Hermes API server, make sure it is enabled:

```bash
echo 'API_SERVER_ENABLED=true' >> ~/.hermes/.env
```

### Step 5: Run Hermes Agent

Start the Hermes gateway:

```bash
hermes gateway run
```

Then send a test request to Hermes:

```bash
python - <<'PY'
from openai import OpenAI

client = OpenAI(base_url="http://localhost:8642/v1", api_key="not-required")
response = client.chat.completions.create(
    model="hermes-agent",
    messages=[
        {
            "role": "user",
            "content": "Use the terminal tool to print the working directory and list 3 files."
        }
    ],
)
print(response.choices[0].message.content)
PY
```

## What Gets Traced

Hermes traces show the full agent execution path in MLflow:

```text
AGENT (agent)
├── LLM (llm.<model>)
├── LLM (api.<model>)
├── TOOL (tool.terminal)
├── TOOL (tool.<name>)
└── ...
```

This makes it possible to see not just the final answer, but also which tools Hermes called, what commands ran, and how long each step took.

## Using AI Gateway Together with Tracing

Hermes works especially well with MLflow when you combine both surfaces:

- **Hermes e2e traces**: sent to MLflow via `hermes-otel` and visible in the Traces UI
- **Gateway usage logs**: captured when Hermes routes model traffic through the [MLflow AI Gateway](/genai/governance/ai-gateway/)

The first shows the full agent workflow. The second shows centralized model routing, budget controls, and guardrails.

## Troubleshooting

**No traces appear in MLflow**

- Verify the MLflow server is reachable at `http://localhost:5000/v1/traces`
- Verify `x-mlflow-experiment-id` points to an existing experiment
- Restart Hermes after editing `~/.hermes/config.yaml` or the plugin config

**Hermes runs but tool spans are missing**

- Confirm the task actually invoked a tool
- Check that `hermes_otel` is enabled under `plugins.enabled`
- Make sure the plugin package is installed into the same environment Hermes runs from

**You only see gateway logs, not full agent traces**

- Gateway logs capture model requests
- Full Hermes agent traces require `hermes-otel` exporting spans directly to MLflow
